\subsubsection*{Grafisches Interface für Linux}
Auch unter Linux steht das Tool \textit{Vidalia} als GUI für Tor zur Verfügung. Das Programm benötigt Qt 4.1 oder höher. Die Sourcen stehen auf der Downloadseite \href{http://tor.eff.org/dist/vidalia-bundles/}{http://tor.eff.org/dist/vidalia-bundles/} zur Verfügung und werden mit dem üblichen Dreisatz installiert.\\

Eine Alternative ist das KDE-Programm \textbf{TorK}. Neben den Funktionen von Vidalia bietet es auch die Möglichkeit, viele Optionen einzustellen sowie diverse Programme für die Nutzung von Tor zu konfigurieren (Konqueror, Kopete, KSirc, SSH-Client). Es steht bereit unter \href{http://www.kde-apps.org/content/show.php?content=39442}{www.kde-apps.org/content/show.php?content=39442}\\

TorK benötigt für die Übersetzung das Entwickler-Packet \textit{kdebase-dev} sowie die davon abhängigen Packete. Unter Debian/GNU Linux wird außerdem das Packet \textit{libkonq4-dev} benötigt. Nach dem Download ist das Archiv zu entpacken und als User root mit dem üblichen Dreisatz zu installieren:

\begin{verbatim}
   # configure --prefix=`kde-config --prefix`
   # make && make install
\end{verbatim}

Das Kommando \textit{kde-config --prefix} zur Ermittlung des KDE-Verzeichnisses ist in Backticks einzuschließen.\\

Die Installationsroutine richtet einen Menüpunkt in der Programmgruppe \textit{Internet} ein. Nach dem ersten Start führt ein Wizzard in drei Schritten durch eine kurze Konfiguration. In den ersten beiden Schritten werden die Programme Tor und Privoxy abgefragt und in der Regel korrekt erkannt. Im dritten Schritt (Bild \ref{abb:tork_first}) wird das Verhalten von Tor festgelegt. Um die Möglichkeiten der grafischen Konfiguration zu nutzen, sollte in der Drop-Down-Box der Punkt \textit{Let me configure Tor} gewählt werden.\\

Im Anschluss können einige Einstellungen für Tor in der Konfiguration angepasst werden. In der Sektion \textit{My Client} kann ganz unten die Tor-Server Funktionalität abgeschaltet werden. Diese Option ist zu empfehlen, wenn der eigene Rechner hinter einem NAT-Gateway oder Firewall nicht vom Internet erreichbar ist.\\

In der Sektion \textit{Konqueror} können die Einstellungen zur Sicherheit angepasst werden. Tork kann diese Optionen mit einem Knopfdruck in der Toolbar zusammen mit der Proxy-Umschaltung aktivieren.\\

Ein Klick auf die grüne Zwiebel in der Toolbar startet den Onion Router. Im Systray erscheint ebenfalls eine grüne Zwiebel. Ein Klick auf das Kreuz daneben beendet den Onion-Router. Wird TorK genutzt, sollte Tor immer vom GUI gestartet und beendet werden. Zwar erkennt TorK eine laufende Instanz des Onion-Routers, ändert in diesem Fall aber sie Konfiguration selbständig. Die Einstellungen müssen dann wieder neu zusammengestellt werden.\\

Nachdem der Onion Router zum ersten Mal 5-10 min gelaufen ist und über einen aktuellen Status des Netzwerkes verfügt, kann man ihn wieder beenden und in den Einstellungen in der Sektion \textit{My Network View} bevorzugte Server als Exit-Knoten festlegen. Wählt man hier schnelle Server aus Mitteleuropa, kann die Geschwindigkeit verbessert werden.\\

TorK bietet auch die Möglichkeit, mehrere Anwendungen für die Nutzung des Onion Routers zu konfigurieren oder so zu starten, dass die Daten über das Tor-Netzwerk geleitet werden. Dieses Feature findet man unter \textit{Anonymize}.\\

Die Option \textit{Anonymous Konqueror} ändert die Proxy-Einstellungen im Kontrollzentrum und die Einstellungen zur Sicherheit (Java und JavaScript deaktivieren, Cache abschalten usw). Sie wirkt auf alle KDE-Programme, die diese Einstellungen nutzen (Quanta, Akregator usw.).\\

Leider werden beim Abschalten von TorK die Default-Einstellungen von Konqueror für Java, Javascript, Cache, Browserkennung und Proxy wieder hergestellt. Man muss nach der Nutzung von TOR diese Einstellungen für spurenarmes Surfen immer wieder anpassen oder einen anderen Browser für das Surfen ohne TOR nutzen.
