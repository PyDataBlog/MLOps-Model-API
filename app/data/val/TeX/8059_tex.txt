\section{Raspberry Pi}

O pequeno computador Raspberry Pi começou a ser desenvolvido 
no começo de 2012. Em 2013 foi feito o seu lançamento para 
o publico comum. É bastante conhecido pelo pequeno tamanho
, atribuido ao processador ARM7 \cite{rasparm} que concentra
as principais funções, em especial a unidade GPU, possibilitando
a execução de filmes em alta definição, 720p. 

A fundação \cite{rasp} é responsável pelo divulgação, treinamento
e projetos sociais que envolvem o Raspberry Pi. Em conjunto com
grandes distribuidores, \textit{Newark element 14}, \textit{RS Components}
e \textit{Egoman}, esta pequena placa pode ser comercializada
com preços baixos, nos tempos atuais, 50 dolares a versão B.
Versão B, esta a mais recente e com ram de 512 Mb.

O processador arm utilizado é ARM1176JZF-S podendo rodar a 700 MHz
com, com VideoCore IV GPU \cite{videocore}. A versão básica do raspberry
vem com memória RAM de 256 MB, já a versão B, a memória passa a ser
512 MB. A Memória ROM, usada para armazenar o sistema operacional, 
softwares, dados, é via cartão de memória SD. 

Maior parte da comunicação com o mundo exterior é dada via comunicação
USB e um barramento de pinos, que disponibiliza uma comunicação I2C, 
SPI e Serial.

\section{Periféricos}

Uma das principais funções do computador embarcado no projeto lavagem a seco
é apresentar a imagem de um webcam em uma tela, para que o operador possa
visualizar, em momento oportuno, a situação da lavagem do(s) isolador(es).

O display deve ser compatível para o sistema GNU/Linux, onde 
o ser driver deve ser suportado. Em outras palavras, o kernel \cite{kernel} deve 
esta compilado com suporte ao display para que o mesmo funcione. 
Se ele não foi compilado, este drive deve ser disponibilizado 
como ``módulo'', para que funcione corretamente.

O maior desafio aqui é encontrar no mercado, um display que satisfaça 
as necessidades do projeto e seja compatível com o GNU/Linux.

O menor tamanho para o display aqui considerado, será de 7 polegadas.
Integrado com um sistema TouchScreen, para o manuseio do software 
pelo operador.
