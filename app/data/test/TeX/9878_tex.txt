\section{Conception}



\subsection{Description textuelle du modèle} 

\begin{itemize}
	\item Les jeux appartiennent à plusieurs catégories.
	\item Les jeux possèdent des succès.
	\item Les jeux peuvent être en promotion.
	\item Les utilisateurs peuvent acheter ces jeux.
	\item Les utilisateurs peuvent gagner des succès en jouant.
	\item Les utilisateurs peuvent publier des tests de jeux.
	\item Chaque jeu et chaque catégorie est associé à un forum de discussion, où les utilisateurs peuvent échanger.
\end{itemize}

\subsection{Table de données}
\paragraph{}
Il nous reste à définir quelles tables objets-relationnelles devront être créées :

\begin{itemize}
	\item User
	\item Game
	\item Category
	\item Achievement
\end{itemize}

\paragraph{}
\underline{Remarques : } On crée une table Achievement car le type User\_t, implémenté dans la table User, a pour attribut une liste de références vers des achievements. Si les achievements n'avaient pas eu leur propre table et étaient simplement contenus dans Game\_t, il aurait été impossible de les référencer car ils seraient seulement des attributs.
